How to use #strategy-feed.

You don't need commands or special syntax. Just post naturally — TARS reads the channel every day at 5 PM and figures out the rest.

✅ Marking an action complete

Post naturally in the channel. TARS recognises completion language and matches it to your open actions.

Billing integration is done ✅
→ TARS matches to your open action and confirms via DM
Finished the Google/Apple compliance review
→ TARS sends confirmation + updates the tracker
Sorted — contract sent to Teya
→ Matched, confirmed, tracker updated same day

Keywords TARS listens for: done · completed · finished · closed · delivered · sorted · wrapped up · shipped · sent · confirmed · resolved · handled

💬 What TARS does next
1
Clear match found — TARS DMs you: "✅ Got it Simon — S08-A03 marked complete. Nice work."
2
Tracker updated — Action_Tracker.xlsx is updated in Google Drive that same evening.
3
Channel acknowledgement — TARS posts once to #strategy-feed if completions were found.
4
If unclear — TARS DMs you asking which action you mean. No guessing, no flagging to others.
📌 Posting context or updates

Use #strategy-feed to share blockers, progress, or context mid-week. TARS reads all of it and surfaces relevant updates in Thursday's preview and Friday's briefing.

How CASE updates this page.
CASE runs every Monday morning. Here is the workflow for connecting real data.
Step 1 — Data input
Prepare a weekly metrics file

Each week, CASE reads a case_data.json file in the GitHub repo. This file contains all the metrics in a structured format.

{ "lastUpdated": "2026-03-23",
  "summary": { "totalUsers": 45230, ... },
  "weeklyRegistrations": [...],
  "topCountries": [...],
  "supportTickets": {...} }
Step 2 — CASE pushes the update
Automatic weekly deployment

Sends summary — Slack message + email with key numbers to leadership
